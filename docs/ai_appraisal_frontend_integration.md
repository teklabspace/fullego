# AI Appraisal — Frontend Integration Guide

Base URL prefix for all endpoints: `/api/v1/assets`

All requests require a valid `Authorization: Bearer <token>` header.

---

## Overview

The AI appraisal is triggered by POSTing to the appraisals endpoint with `appraisal_type: "API"`. The response comes back immediately (no polling needed) and always includes an estimated value — even when the asset has incomplete data. If data is thin, the AI sets confidence to `low` and lists what's missing so the user can fill it in.

---

## Endpoints

### 1. Trigger an AI Appraisal

```
POST /api/v1/assets/{asset_id}/appraisals
```

**Request body**

```json
{
  "appraisal_type": "API"
}
```

> For concierge/human appraisals use a different `appraisal_type` value (`"Concierge"`, `"Standard"`, etc.). This doc covers the AI path only.

|------|------|

```json
{
```

```
GET /api/v1/assets/{asset_id}/appraisals
```

### 4. Check AI Usage Quota

```
GET /api/v1/assets/ai/usage
```

Call this before showing the "Get AI Appraisal" button so you can warn or disable it when the quota is exhausted.

**Response — HTTP 200**

```json
{
  "plan": "monthly",
  "period": "2026-06",
  "ai_appraisals": {
    "limit": 25,
    "used": 3,
    "remaining": 22
  },
  "ai_reviews": {
    "limit": 25,
    "used": 1,
    "remaining": 24
  }
}
```

`limit` and `remaining` are `null` for unlimited plans (Annual).

## Appraisal Status Values

The `status` field on the appraisal record tells you what happened and how to present the result.

| Status | Meaning | Has estimate? | UI guidance |
|--------|---------|:---:|-----------|
| `ai_appraised` | AI generated a confident estimate | ✅ | Show full result normally |
| `needs_more_information` | Estimate generated but key data was missing | ✅ | Show estimate with a banner: "Accuracy can improve — see missing fields below" |
| `professional_appraisal_recommended` | AI flagged this asset as needing a human expert | ✅ | Show estimate + a callout recommending a professional appraisal |
| `appraisal_failed` | AI service error | ❌ | Show error state; `notes` contains the error message |
| `pending` | Human appraisal requested, awaiting assignment | ❌ | Show "Pending" state (concierge flow) |
| `completed` | Human appraisal completed | ✅ | Legacy status, treat same as `ai_appraised` |

## The `ai_result` Object — Field Reference

Both `response.ai_result` (top-level on POST) and `response.data.ai_data` (stored on the record) contain the same fields.

| Field | Type | Description |
|-------|------|-------------|
| `estimated_value` | number | Single best-estimate market value |
| `value_range_low` | number | Low end of plausible range |
| `value_range_high` | number | High end of plausible range |
| `currency` | string | 3-letter ISO currency code (e.g. `"USD"`) |
| `confidence` | `"low"` \| `"medium"` \| `"high"` | AI's confidence given available data |
| `appraisal_summary` | string | 2-4 sentence summary of the finding |
| `key_value_drivers` | string[] | Factors supporting the estimated value |
| `risk_factors` | string[] | Factors that could reduce or complicate value |
| `missing_information` | string[] | Data fields absent that would improve accuracy — empty `[]` if complete |
| `recommended_documents` | string[] | Documents that would strengthen the appraisal |
| `suggested_next_step` | string | Actionable recommendation for the asset owner |
| `professional_appraisal_needed` | boolean | `true` when the AI recommends a certified professional appraisal |
| `disclaimer` | string | Must be displayed alongside the estimated value |
| `model` | string | AI model used (e.g. `"claude-opus-4-8"`) |

## UI Implementation Notes

### Confidence badge

Map `confidence` to a colour:

```
"high"   → green badge
"medium" → amber badge
"low"    → red/orange badge + show missing_information list
```

### Disclaimer

Always render `ai_result.disclaimer` (or `ai_data.disclaimer`) directly below the estimated value. Do not paraphrase it.

### Missing information banner

When `missing_information.length > 0`, show a banner listing the missing fields so the user knows they can improve the estimate by updating the asset. Example wording:

> **To improve accuracy, add:** Serial number, original purchase receipt, gemological certificate

### Professional appraisal callout

When `professional_appraisal_needed === true` (status will be `professional_appraisal_recommended`), show a callout:

> **A certified appraisal is recommended for this asset.** The AI estimate is provided for reference, but a professional appraisal is advised before insuring or selling.

### Quota-aware button

Before rendering the "Get AI Appraisal" button, call `GET /api/v1/assets/ai/usage`. If `ai_appraisals.remaining === 0` (or `limit` is not null and `used >= limit`), disable the button and show:

> You've used all AI appraisals for this month. Upgrade your plan or wait until next month.

For unlimited plans (`remaining === null`), always enable the button.

## Complete Trigger Flow (pseudocode)

```js
async function runAIAppraisal(assetId) {
  // 1. Optional: check quota first
  const usage = await api.get(`/api/v1/assets/ai/usage`)
  if (usage.ai_appraisals.remaining === 0) {
    showQuotaError(usage)
    return
  }

  // 2. Trigger appraisal
  setLoading(true)
  try {
    const res = await api.post(`/api/v1/assets/${assetId}/appraisals`, {
      appraisal_type: "API"
    })

    const { data, ai_result } = res

    // 3. Render based on status
    switch (data.status) {
      case "ai_appraised":
        showAppraisalResult(ai_result)
        break
      case "needs_more_information":
        showAppraisalResult(ai_result, { showMissingBanner: true })
        break
      case "professional_appraisal_recommended":
        showAppraisalResult(ai_result, { showProCallout: true })
        break
      case "appraisal_failed":
        showError("Appraisal could not be completed. Please try again.")
        break
    }
  } catch (err) {
    if (err.status === 403) showQuotaError()
    else if (err.status === 503) showServiceUnavailable()
    else showGenericError()
  } finally {
    setLoading(false)
  }
}
```

## Notes for the Frontend Team

- The POST response returns `ai_result` at the top level for immediate use. The same data is also stored in `data.ai_data` so it's available when fetching the appraisal later from the list or single-appraisal endpoints.
- The AI call takes 3-8 seconds typically. Show a loading spinner during this time — do not poll; the response comes back in a single request.
- All monetary values from the API are strings (Decimal serialization). Parse with `parseFloat()` or your preferred decimal library before arithmetic.
- `missing_information` is always an array (never null). Safe to call `.length` directly.
- `key_value_drivers`, `risk_factors`, `recommended_documents` are also always arrays.
